# 推送消息接收问题诊断指南

## 🔍 问题排查步骤

### 1. 检查应用状态

#### 1.1 应用是否在前台/后台
- **前台**：应用正在使用中，应该能收到透传消息
- **后台**：应用在后台运行，可能收不到透传消息，但能收到通知消息
- **被杀死**：应用完全关闭，只能收到通知消息

#### 1.2 检查应用权限
```bash
# 在设备设置中检查：
# 设置 > 应用 > 您的应用 > 权限
# 确保以下权限已开启：
# - 通知
# - 网络访问
# - 后台运行（如果支持）
```

### 2. 检查网络连接

#### 2.1 网络状态
- 确保设备有稳定的网络连接
- 检查是否在WiFi或移动网络下
- 某些网络环境可能阻止推送连接

#### 2.2 防火墙/代理
- 公司网络可能有防火墙限制
- 个人网络可能有代理设置
- 尝试切换到移动网络测试

### 3. 检查个推配置

#### 3.1 应用配置
```typescript
// 检查 config/pushConfig.ts 中的配置
```

### 4. 检查CID获取

#### 4.1 CID状态
- 在推送测试页面查看CID是否已获取
- CID为空表示个推SDK未正确初始化
- 尝试点击"刷新CID"按钮

#### 4.2 个推控制台
- 登录个推控制台
- 查看设备是否已注册
- 检查CID是否与设备匹配

### 5. 检查推送类型

#### 5.1 透传消息
- 不会在通知栏显示
- 应用在前台时才能收到
- 需要应用主动处理

#### 5.2 通知消息
- 会在通知栏显示
- 应用在前台/后台都能收到
- 点击通知会触发回调

### 6. 检查原生模块

#### 6.1 编译状态
```bash
# 重新编译应用
cd android
./gradlew clean
./gradlew assembleDebug
```

#### 6.2 日志检查
```bash
# 查看Android日志
adb logcat | grep -E "(GetuiMessageReceiver|MainActivity|GetuiModule)"
```

### 7. 常见问题解决

#### 7.1 应用被杀死
**问题**：应用被系统杀死后收不到透传消息
**解决**：
- 确保应用在后台运行
- 检查电池优化设置
- 添加应用自启动权限

#### 7.2 网络问题
**问题**：网络连接不稳定导致推送失败
**解决**：
- 切换到稳定的网络
- 检查网络权限
- 重启网络连接

#### 7.3 配置错误
**问题**：AppID/AppKey/AppSecret配置错误
**解决**：
- 检查个推控制台配置
- 确认环境配置正确
- 重新配置应用

#### 7.4 权限问题
**问题**：通知权限未授权
**解决**：
- 在设置中开启通知权限
- 重启应用
- 重新授权权限

- 检查送达情况
- 查看错误日志

### 9. 测试步骤

#### 9.1 基础测试
1. 确保应用在前台运行
2. 获取有效的CID
3. 发送透传消息测试
4. 检查控制台日志

#### 9.2 通知测试
1. 发送通知消息
2. 检查通知栏显示
3. 点击通知测试回调
4. 查看应用内处理

#### 9.3 后台测试
1. 将应用切换到后台
2. 发送透传消息（应该收不到）
3. 发送通知消息（应该能收到）
4. 点击通知测试

### 10. 联系支持

如果以上步骤都无法解决问题，请提供以下信息：

1. **设备信息**：型号、系统版本
2. **应用版本**：当前使用的版本
3. **网络环境**：WiFi/移动网络
4. **错误日志**：控制台和Android日志
5. **测试步骤**：详细的操作步骤
6. **期望结果**：应该收到的消息类型
