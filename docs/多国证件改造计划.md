# mock-data-generator.html 多国联动与多证件生成改造

## Context（为什么改）

当前 `mock-data-generator.html` 只支持**中国大陆**：固定输出一列身份证，地址用省/市/区三级 GB/T 2260 行政区划码，姓名/手机号/邮编都是中国规则。

用户需要把它扩展为**多国测试数据生成器**：
- 新增「国家」下拉，默认中国。
- 选择国家后，**手机号、证件号、邮编、省市 code、姓名、地址**全部随国家联动重新生成。
- 证件类型大幅扩展：除身份证外，新增**护照、港澳通行证、台湾通行证、港澳居民来往内地通行证、台胞证**，以及各国本国证件（美国 SSN、日本 MyNumber、韩国居民登录号、新加坡 NRIC、马来西亚 MyKad 等）。
- 每种证件**单独一列**，用户在字段区**多选**当前国家支持的证件类型。
- 覆盖中外常用 **13 国**（含港澳台作为中国地区，不单独成国）。
- 省市编码：中国继续 GB/T 2260 六位码；国外用 **ISO 3166-2**（如 `US-CA`、`JP-13`）。

约束：**保持单文件、无构建**（CLAUDE.md 要求纯前端单文件直接浏览器打开）。所有改动集中在这一个文件。

## 架构决策（4 条核心设计）

1. **`COUNTRIES` 配置字典**：每个国家一个对象，集中声明 name/callingCode/regionSystem/regions/idTypes/fields 开关/gen 策略。加国家 = 加一个条目。
2. **`ID_TYPE_DEFS` 证件注册表**：每个证件 key → `{ label, gen(ctx) }`，校验位算法（Luhn / Mod-11 / 各国专用）封装在各 `gen` 内。国家支持的证件由 `COUNTRIES[country].idTypes` 引用这些 key。
3. **生成派发用「策略对象」per country**：`COUNTRIES[country].gen = { name, phone, postcode, address, regionCodes }`，`genOne` 按 `country` 查表派发。契合现有「每个维度一个 `genXxx`」的风格，避免 `switch(country)` 的 60+ 分支。现有 `genName/genPhone/genPostcode/genAddress` 改名为 `genNameCN/...` 作为 CN 策略被调用，逻辑基本不动。
4. **地址控制器抽象**：引入 `currentAddrModel`（当前地区树引用）+ `rebuildAddressForCountry(country)` 总调度。`SearchSelect` 类零改动（已容器无关、数据驱动）；现有 `onProvinceChange/onCityChange/getCurrentCodes/autoFillPostcode/locateByCode` 的数据源从 `ADDR_DATA` 换成 `currentAddrModel` + 按 `regionSystem` 分支。**不搞两套下拉显隐**，复用同一组下拉，国外时第三级隐藏。

### 统一 areaInfo 形状（CN 与国外共用）
```
{ l1:一级行政区, l2:二级, l3:三级(CN)/null, iso:ISO码(国外)/null, code3:CN区级6位码/null, zip:邮编前缀(国外), area:电话区号(国外) }
```

### 统一字段 key 命名
- 共性字段 key：name/gender/birthday/ethnicity/phone/landline/postcode/provinceCode/cityCode/bankName/bankCard/creditCode/company/email/address
- 证件列 key：`idcol_` + 证件 key（如 `idcol_us_ssn`），避免多国多证件冲突
- 省市 code：CN provinceCode=省6位码(110000)、cityCode=市6位码(440300)；国外 provinceCode=ISO(US-CA)、cityCode=空

### 证件号格式规则（带真实校验位）

- **身份证** `cn_resident`：18 位 GB/T 2260 加权校验。
- **中国护照** `cn_passport`：`E` + 8 位数字。
- **往来港澳通行证** `hk_permit`：`C` + 8 位数字（内地居民赴港澳）。
- **往来台湾通行证** `tw_permit`：`L` + 8 位数字（内地居民赴台）。
- **港澳居民来往内地通行证** `hk_mainland`：`H`/`M` + 8 位数字。
- **台胞证** `tw_mainland`：8 位数字（2015 版）。
- **美国护照** `us_passport`：9 位数字。**SSN** `us_ssn`：`AAA-GG-SSSS`，AAA∉{000,666,9xx}、GG≠00、SSSS≠0000。
- **英国护照** `gb_passport`：9 位数字。**NINO** `gb_nino`：2 字母(避开 D/F/I/Q/U/V/O)+6 数字+1 字母(A-D)。
- **日本护照** `jp_passport`：`M` + 8 位数字。**MyNumber** `jp_mynumber`：12 位 Mod-11（权重 6,5,4,3,2,7,6,5,4,3,2）。
- **韩国护照** `kr_passport`：`M` + 8 位数字。**居民登录号** `kr_rrn`：6 位生日-7 位 + Mod-11。
- **加拿大护照** `ca_passport`：2 字母 + 6 数字。**SIN** `ca_sin`：9 位 Luhn。
- **澳大利亚护照** `au_passport`：2 字母 + 7 数字。
- **德国护照** `de_passport`：`C`/`L` + 8 位数字。
- **法国护照** `fr_passport`：2 字母 + 7 数字。**INSEE** `fr_insee`：15 位（性别+生日+地区+校验键）。
- **新加坡 NRIC/FIN** `sg_nric`：字母(S/T/F/G/M)+7 数字+校验字母。**新加坡护照** `sg_passport`：`E` + 7 数字。
- **马来西亚 MyKad** `my_mykad`：12 位 `YYMMDD-PP-####G`。**马来西亚护照** `my_passport`：1 字母 + 8 数字。
- **泰国身份证** `th_id`：13 位 Mod-11。**泰国护照** `th_passport`：1 字母 + 7 数字。
- **俄罗斯护照** `ru_passport`：10 位数字。**INN** `ru_inn`：12 位双 Mod-11。

## 函数级改动清单（按区块）

### 区块 A — 新增数据结构（插入到 `FIELDS` 之后、`SURNAMES` 之前）
- `COUNTRIES`（13 国）、`ID_TYPE_DEFS`（全部证件）、各国地区数据（ISO3166-2+城市+邮编前缀+电话区号）、各国姓名/街道词库。

### 区块 B — 模块级状态
- `let currentCountry='CN'`、`let currentAddrModel=ADDR_DATA`、`let userFieldPrefs={}`。

### 区块 C — 地址选择器（改造现有）
- `onProvinceChange/onCityChange`：数据源 `ADDR_DATA` → `currentAddrModel`。
- `getCurrentCodes`：按 `regionSystem` 分支，统一返回形状。
- `autoFillPostcode`：CN 走 POSTAL_MAP，国外走国家配置城市 zipPrefix。
- `locateByCode`：CN 维持；国外禁用（#locateWrap 隐藏）。
- `updateAddrPreview`：文案泛化。
- `bindPostcodeInput`：**关键**——`\D` 数字过滤仅纯数字邮编国家；UK/CA 字母数字邮编不过滤，maxlength/inputmode 由 JS 设置。
- 新增 `rebuildAddressForCountry(country)`、`onCountryChange(v)`。

### 区块 D — 生成器（改造 + 新增）
- 重命名 `genName→genNameCN`、`genPhone→genPhoneCN`、`genPostcode→genPostcodeCN`、`genAddress→genAddressCN`。
- 现有 `genIdCard` 收进 `ID_TYPE_DEFS['cn_resident'].gen`。
- `genGender/genBirthday/genEthnicity/genLandline/genBankCard/genCreditCode/genCompany/genEmail` 保留。
- 新增各国 `genNameXX/phoneXX/postcodeXX/addressXX`、`genCompanyFor/genEmailFor`、全部证件 gen。

### 区块 E — 组装
- 重构 `genOne(settings)`：派发式，证件列键名 `idcol_<k>`。
- 改 `getCustomSettings()`：加 `country`，`areaInfo` 形状泛化。

### 区块 F — UI 字段区（解耦 `FIELDS` 硬编码）
- 新增 `computeFields(country)`：动态产出字段清单。
- 改 `buildCheckboxes`→`renderFieldCheckboxes(country,prefs)`：动态字段 + 证件独立多选 + 全选联动。
- 改 `getSelectedFields`：返回 `{key,label}` 对（不依赖全局 FIELDS）。
- 改 `renderTable/exportCSV`：去掉 `FIELDS.find`。

### 区块 G — HTML（最小改动）
- 左面板顶部加 `<select id="countrySelect">`。

- 邮编输入框去掉硬编码 maxlength/inputmode，改 JS 设置。
- 定位输入框外包 `<div id="locateWrap">`。

## 边界处理（重点坑）

1. **港澳台无大陆身份证**：`ID_TYPE_DEFS['cn_resident'].gen` 检查地区码前缀 81/82/71，命中留空 + genOne 收集告警 + 生成结束 toast。**在 gen 阶段判断**。
2. **国外裁剪中国特有字段**：`COUNTRIES[country].fields` 开关——国外 ethnicity/landline/bank/creditCode 全 false，不出列、不写键。
3. **UK/CA 字母邮编**：bindPostcodeInput 数字过滤按国家分支。
4. **跨国家切换状态**：共性字段勾选跨国家保留；证件勾选重置为该国默认主证件；不支持字段静默移除。
5. **生日**：统一 `YYYY-MM-DD`。
