### 2.3 流程状态定义

| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| DRAFT | 草稿 | 模板配置中 |
| PENDING_QUALITY | 待资质初审 | 等待资质审核 |
| QUALITY_FAILED | 资质审核不通过 | 资质审核未通过 |
| PENDING_REVIEW | 待评审 | 等待评审 |
| WRITTEN_REVIEWED | 书面评审完成 | 书面评审已完成 |
| FIELD_INSPECTED | 实地考察完成 | 实地考察已完成 |
| REVIEW_FAILED | 评审不通过 | 评审未通过 |
| PENDING_FUND | 待发放补助 | 等待资金发放 |
| COMPLETED | 已完成 | 流程完成 |
| TERMINATED | 已终止 | 流程终止 |

### 2.4 流程异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 模板配置失败 | 回滚到上一步，提示错误信息 |
| 资质审核超时 | 系统自动提醒，可升级处理 |
| 实地考察无法进行 | 记录原因，可申请延期或视频考察 |
| 评审意见不一致 | 组织评审会议，集体决策 |
| 资金发放失败 | 记录失败原因，联系银行处理 |

### 4.2 规则配置流程

#### 步骤1：创建规则
| 操作 | 说明 |
|------|------|
| 入口 | 数据监测 > 数据质量管理 > 新增规则 |
| 填写内容 | 规则名称、规则类型、目标表、目标字段、规则表达式 |
| 规则类型选择 | 完整性/准确性/一致性/时效性 |

#### 步骤2：配置表达式
| 规则类型 | 表达示例 | 说明 |
|---------|---------|------|
| 完整性 | `field !== null && field !== ''` | 字段不能为空 |
| 准确性 | `/^\d{11}$/.test(phone)` | 手机号格式校验 |
| 一致性 | `parentTable.find(item => item.id === field)` | 关联数据存在 |
| 时效性 | `(Date.now() - new Date(field)) / 86400000 <= 7` | 数据7天内更新 |

#### 步骤3：规则测试
| 操作 | 说明 |
|------|------|
| 测试数据源 | 选择测试数据表 |
| 执行测试 | 运行规则检查 |
| 查看结果 | 查看通过率、失败记录 |

#### 步骤4：规则启用
| 操作 | 说明 |
|------|------|
| 确认启用 | 确认规则配置正确 |
| 设置优先级 | 设置规则执行优先级 |
| 启用规则 | 规则生效 |

### 4.3 质量问题处理流程

#### 问题级别定义

| 级别 | 名称 | 响应时间 | 处理要求 |
|------|------|---------|---------|
| P0 | 严重 | 1小时内 | 立即处理 |
| P1 | 重要 | 4小时内 | 当日处理 |
| P2 | 一般 | 24小时内 | 本周处理 |
| P3 | 提示 | 7天内 | 择机处理 |

#### 问题处理步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 问题发现 | 质量检查发现问题 |

| 3 | 问题分派 | 分配给相应处理人 |
| 4 | 问题处理 | 处理人分析并修复问题 |
| 5 | 结果验证 | 验证问题是否解决 |
| 6 | 问题关闭 | 关闭问题记录 |

## 7. 异常处理流程

### 7.1 系统异常处理

| 异常类型 | 处理方式 | 通知对象 |
|---------|---------|---------|
| 系统宕机 | 自动重启/人工介入 | 系统管理员 |
| 数据库连接失败 | 重连/切换备用 | 系统管理员 |
| 接口超时 | 重试机制 | 调用方 |
| 文件上传失败 | 提示用户/记录日志 | 用户 |

### 7.2 业务异常处理

| 异常类型 | 处理方式 | 通知对象 |
|---------|---------|---------|
| 审核超时 | 自动提醒/升级 | 审核员+上级 |
| 资金发放失败 | 记录原因/重试 | 财务人员 |
| 数据异常 | 预警通知/人工处理 | 数据管理员 |
