## 9. 数据监测模块（核心功能）

### 9.1 数据质量管理

#### 9.1.1 质量规则配置
| 功能点 | 说明 |
|--------|------|

| 规则类型 | 完整性规则、准确性规则、一致性规则、时效性规则 |

#### 9.1.2 质量规则表单
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ruleName | String | 是 | 规则名称 |
| ruleCode | String | 是 | 规则编码 |
| ruleType | String | 是 | 规则类型 |
| targetTable | String | 是 | 目标表名 |
| targetField | String | 是 | 目标字段 |
| ruleExpression | String | 是 | 规则表达式 |
| severity | String | 是 | 严重级别：高/中/低 |
| isEnabled | Boolean | 是 | 是否启用 |
| description | Text | 否 | 规则描述 |

#### 9.1.3 质量规则表达式示例
```javascript
// 完整性规则：字段不能为空
{
  "type": "completeness",
  "expression": "field !== null && field !== ''"
}

// 准确性规则：邮箱格式校验
{
  "type": "accuracy",
  "expression": "/^[\\w-]+(\\.[\\w-]+)*@[\\w-]+(\\.[\\w-]+)+$/.test(field)"
}

// 一致性规则：关联数据存在
{
  "type": "consistency",
  "expression": "relatedTable.find(r => r.id === field) !== undefined"
}

// 时效性规则：数据更新时间
{
  "type": "timeliness",
  "expression": "(Date.now() - new Date(field)) / 86400000 <= 7"
}
```

| startTime | String | 否 | 开始时间 |
| endTime | String | 否 | 结束时间 |
