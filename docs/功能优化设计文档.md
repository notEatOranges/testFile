> 工程目录：`AAAAAA/`（微信小程序 + 云开发，TDesign 组件库，Skyline 渲染）
> 范围：除「聊天」外全部功能的优化 / 重构，外加游戏大厅。

## 0. 现状与改造原则

### 0.1 现有数据层（不要破坏的契约）
- `utils/store.js`：基于云数据库 `kv` 集合（文档 `{room, path, value, ts}`，唯一键 `(room, path)`）的实时抽象。`room = coupleId`。
  - `Store.onValue(path, cb)` / `onList(path, cb)`（实时）、`getOnce(path)`、`set/update/push/remove/transaction`（写，经 `kvWrite` 云函数）。
  - 写后 `emit()` 触发相交路径订阅回拉，保证「自己写自己立即可见」。
- `utils/user.js`：身份层。`user.getUser() → {openid, nick, avatar, coupleId, role}`，`saveProfile({nick,avatar})`、配对相关。
- `utils/room.js`：心跳/在线。`room.getRole()`(boy/girl)、`getPeer()`、`join()/leave()`、`onPeerPresence()`。
- `utils/notify.js`：订阅消息额度。模板 `TMPL_CHAT`、`TMPL_ANNIV`。
- 推送链路：`sendMsg`（聊天，写 kv + 推 TMPL_CHAT）、`anniversaryReminder`（每日 9:00 定时，扫 `anniversary/events`，推 TMPL_ANNIV）。

### 0.2 铁律（来自 `AAAAAA/CLAUDE.md`）
- **UI 禁止用 emoji 当图标/装饰，一律 `t-icon`**（用户选择的心情 emoji 属于产品数据，不在此列）。
- 表单/按钮用 TDesign；原生 `<input>` 用显式 `height + line-height`，不要靠 padding 撑高。
- 业务页 json 设 `"navigationStyle": "default"`。
- 主题：页面根节点 `class="theme-{{theme}} love-page"`，配色走 `love-theme.wxss` CSS 变量；Canvas 取色用 `utils/themes.js`。

### 0.3 本轮全局改造：网名体系（功能 5 + 6 的地基）
现状到处用 `roleFull`(男生/女生)、`roleName`(他/她) 称呼对方。改为**统一用网名 + 头像**。

- 新增 `utils/ident.js`（身份展示层，集中管理「我/对方的网名头像」，替代散落各页的 roleFull）：
  - `ident.bind(page)`：在页面 `data` 注入 `{ role, peer, myNick, myAvatar, peerNick, peerAvatar }`；订阅 `profile/{peer}` 实时拿对方网名头像；并把「我自己」的 profile 写进 `profile/{role}`（对方读得到）。
  - `ident.teardown(page)`：在 `onUnload` 退订。
  - 无网名时回退为 `'我' / 'ta'`（**不再出现「男生/女生」字样**）。
- 所有功能页（mood/days/wishlist/truthbox/me/main）改用 `ident` 提供的网名头像展示。
- `utils/util.js` 的 `roleName/roleFull/peerRole` 保留（内部角色逻辑仍需 boy/girl），但**展示层不再调用 roleFull**。

### 0.4 通用 UI 组件
- 统一卡片基类 `love-card`（已存在于全局 wxss）。
- 本轮新增可复用样式：分类标签 `cat-chip`、进度条 `mini-progress`、堆叠卡片 `stack-card`、空状态 `empty-state`（均写进各页或全局 wxss，遵循现有 CSS 变量）。

## 1. 功能 1：今日心情（重构）

**诉求**：自己 + 对方都能看；卡片式；历史心情；自定义表情；背景图；背景图与文字颜色混合（防看不清）。

### 1.1 数据模型（kv path）
- 每日心情（保持向后兼容，扩展字段）：
  ```
  mood/{YYYY-MM-DD} = {
    boy: { emoji, label, whisper, bg, textColor, ts },
    girl:{ ...同上 }
  }
  ```
  - `bg`：背景图 cloud fileID（空 = 无背景，走主题卡片色）。
  - `textColor`：与背景图混合后算出的文字色（`#1a1a1a` 或 `#ffffff`），见 1.3。
  - `label`：自定义表情可不带 label，存空串。
- 历史心情：无需额外索引，按前缀查（见 1.4）。

### 1.2 今日页 UI
- 顶部两张并排卡片：**我的今日** / **ta 的今日**（头像 + 网名 + 大表情 + label + 悄悄话）。
- 有背景图时：卡片以背景图为底，叠半透明 scrim（`linear-gradient(rgba(0,0,0,.28), rgba(0,0,0,.45))`），文字用 `textColor`。
- 表情选择区：预设网格（现有 `MOODS`）+ 一个「自定义」入口 → 弹出输入框（允许输入任意 emoji/文字，最多 4 字）。
- 悄悄话 `t-textarea`（沿用）。
- 背景图入口：`t-icon name="image"`，`wx.chooseMedia` 选图 → 上传云存储 → 计算 textColor → 保存。
- 底部「历史心情」入口 → 跳历史页。

### 1.3 背景图与文字颜色混合算法
新增 `utils/colorMix.js`：
1. 把背景图画到 50×50 离屏 canvas（`wx.createOffscreenCanvas` / 2d），`canvasGetImageData`。
2. 取像素平均 RGB → 亮度 `L = 0.299R + 0.587G + 0.114B`。
3. `L > 140` → 文字深色 `#1a1a1a`、scrim 用浅色；否则文字 `#ffffff`、scrim 用深色。
4. 额外保证：scrim 永远叠加，确保任何图都可读（混合 + 兜底双保险）。
- 结果 `textColor` 随 `bg` 一起存，渲染端零计算。

### 1.4 历史心情页 `packageFunc/mood/history`
- `Store.getPrefix('mood/')`（**store.js 新增方法**：`where({room}) + db.RegExp({regexp:'^mood/'})` 一次性拉全部 mood 文档，合并为 `{date: value}`）。
- 按日期倒序列出卡片流；每张卡片展示当天两人表情 + 悄悄话 + 缩略背景。
- 下拉刷新重拉（历史非实时，无 watch）。

## 2. 功能 2：纪念日（重构 + 推送）

**诉求**：从毛坯升级为成熟方案；按年循环；分类；倒数/正数；照片；接入消息推送。

### 2.1 数据模型
```
anniversary = { startDate }                         // 在一起起始日（正数天数）
anniversary/events = {
  id: {
    title, date,            // 'YYYY-MM-DD'
    type,                   // 'anniversary'|'birthday'|'countdown'|'first'|'festival'
    recurrence,             // 'once'|'yearly'|'monthly'
    note, photo,            // 备注、照片 cloud fileID（可空）
    advanceDays,            // 提前提醒天数，如 [0,1,3,7]；默认 [0,1]
    createdBy, ts
  }
}
```

### 2.2 页面 UI（卡片化 + 分类 + 进度）
- Hero：在一起第 N 天（保留），副标显示起始日。
- 「即将到来」分组：按下一次发生日升序，展示「还有 X 天 / 就是今天 / X 天前」。周年/生日按循环计算下次发生。
- 卡片：分类图标（t-icon：`heart`/`gift`/`time`/`star`/`calendar`）+ 标题 + 日期 + 备注 + 照片缩略 + 倒数大数字。
- 添加表单：标题、日期、分类 picker、循环 picker（单次/每年/每月）、提前天数多选、备注、照片。
- 删除沿用 modal。

### 2.3 下次发生日计算（客户端展示 + 云函数推送共用）
新增 `utils/anniv.js`：
- `nextOccur(date, recurrence)`：yearly → 取 date 的月日，落在今年；若已过则明年。monthly → 本月对应日，过则下月。once → date 本身。
- `daysToNext(event)`：`daysUntil(nextOccur(...))`。
- 展示：负数=已过（X 天前，单次）/ 0=今天 / 正数=还有 X 天。

### 2.4 推送改造 `cloudfunctions/anniversaryReminder/index.js`
现状只按 `e.date` 原值算 `daysUntil`，**循环事件明年不触发**。改为：
- 对每个 event 用 `nextOccur(date, recurrence)` 算下次发生日，再 `daysUntil`。
- 用 event 自带 `advanceDays`（缺省 `[0,1]`）判定是否命中提醒。
- 其余（扫 couples → 读 `anniversary/events` → 给双方发 TMPL_ANNIV）逻辑不变。
- `config.json` 触发器保持每日 9:00 CST。

## 3. 功能 3：心愿清单（重构）

**诉求**：从潦草升级；分类；打卡（照片 + 感受）；进度；双人协作。

### 3.1 数据模型
```
wishlist = {
  id: {
    text, category,          // 'go'(想去)|'do'(想做)|'eat'(想吃)|'achieve'(想完成)
    done, photo, note,       // 完成图、完成感受

}
```

### 3.2 页面 UI
- 顶部分类 Tab：全部 / 想去 / 想做 / 想吃 / 想完成；Tab 旁进度条 `已完成 X / 总 Y`。
- 心愿卡片：分类色条 + 文本 + 完成态；已完成展示照片缩略 + 完成者网名头像 + 感受。
- 完成时弹半屏表单：上传照片（可选）+ 写一段感受（可选）→ 完成动画/庆祝 toast。
- 底部固定添加栏：文本 + 分类 picker + 添加按钮。
- 谁提出（createdBy 网名）、谁完成（completedBy 网名）用网名展示。

## 4. 功能 4：真心话（重构）

**诉求**：现在是「单题 + 坏的」；要堆叠卡片、多题并存、答题前可改、对方答后锁定、互看答案、分类题库、联网找题。

### 4.1 数据模型（多题并存）
```
truthbox/questions = {
  id: {
    text, category,          // 'icebreak'(破冰)|'sweet'(甜蜜)|'deep'(深度)|'adventure'(冒险)|'wilder'(脑洞)
    createdBy, createdTs,
    answers: { boy:{text,ts}, girl:{text,ts} }   // 任一方未答则缺键
  }
}
```
- 任意一方可加题（手动输入 或 从题库抽）。
- **锁定规则**：我的答案在「对方尚未作答」时可改；一旦 `answers[peer]` 存在 → 我的答案只读（不可改）。
- **互看规则**：我在 `answers[me]` 存在后即可看到 `answers[peer]`（双方都先落笔才互相公开）。

### 4.2 页面 UI（堆叠卡片）
- 顶部题库入口：「抽一题」（按分类）、「我来出题」（输入框）。
- 主体**堆叠卡片区**：卡片像一摞扑克，最上层是最新/未答完的题，下层露出 1~2 张边缘做层叠感（`transform: translateY + scale + 透明度`）。点击上层卡片展开为答题视图（我的答案输入/只读 + 对方答案展示/遮挡占位）。
- 答题视图状态：
  - 我未答 → 输入框可编辑 +「提交」。
  - 我已答、对方未答 → 我的答案可改 +「对方还没回答，ta 回答后双方答案公开」+ 对方占位遮挡。
  - 双方都答 → 双答案并排展示，我的答案锁定只读。
- 全部答完的题沉到底部「默契回顾」分组。

### 4.3 分类题库 + 联网找题
- `utils/util.js` 的 `TRUTH_QUESTIONS` 扩成**分类题库** `TRUTH_BANK = { icebreak:[...], sweet:[...], deep:[...], adventure:[...], wilder:[...] }`，每类 15+ 题（参考 Vogue 315 题 / 知乎情侣 100 条，见文末来源）。
- 新增云函数 `truthQuestions`：服务端维护更大题库，`event = {category|'random'}` 返回若干题。客户端「抽一题」优先调云函数（**联网找题**），失败回退本地 `TRUTH_BANK`。这样题库可后续扩且免发版。

## 7. 功能 7：游戏大厅 + 三款游戏

### 7.1 大厅 `packageFunc/lobby/lobby`
- 卡片网格入口：2048、俄罗斯方块、你画我猜（各一张带 t-icon 的卡）。
- `main.js` 的 `lobby` 卡片 `ready:true`，`app.json` 注册 lobby 与各游戏子包页。

### 7.2 2048 `packageFunc/games/game2048`
- Canvas 2d 实现，触屏滑动操作。主题色取 `themes.getTheme()`。
- 最高分存 `games/2048/best/{role}`（各自最高分，互看）。

### 7.3 俄罗斯方块 `packageFunc/games/tetris`
- Canvas 实现，下落/旋转/消行/计分。主题色同上。
- 最高分存 `games/tetris/best/{role}`。

### 7.4 你画我猜 `packageFunc/games/drawguess`（双人回合制）
- **回合制**（避开高频轨迹写入限流）：一人当画手抽词 → 在 canvas 画 → 完成/提交（把**最终图片 fileID** 或节流后的稀疏笔画上传）→ 写 `games/dg/round = {word, drawer, drawing, guess, state, winner}`。对方经 `watch` 看到 → 输入框猜 → 命中即本轮胜，交换画手。
- 词库 `utils/dgWords.js`（名词 + 提示分类）。
- 计分 `games/dg/score/{role}`。

### 7.5 子包结构
新增子包 `packageGames`（root: `packageFunc/games`）或直接放 `packageFunc` 下。为避免主包过大，建议新子包：
```
subpackages: [
  { root:'packageFunc', pages:[ chat, mood, days, wishlist, truthbox, lobby, mood/history ] },
  { root:'packageGames', pages:[ game2048/game2048, tetris/tetris, drawguess/drawguess ] }
```
