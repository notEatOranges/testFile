## ⚙️ 工作流铁律（每次任务都必须遵守，写进了 CLAUDE.md 与我的长期记忆）

1. **先读设计文档**：[功能优化设计文档.md](./功能优化设计文档.md) 是单一事实来源。
2. **逐功能点实现**：一个功能点 = 一次 `git commit`，提交信息写清「做了什么 / 改了哪些文件 / 为什么」。
3. **完成即记录**：每提交一次，立刻回到本文件打勾 + 填写「提交记录」一行（哈希、日期、要点）。
4. **接力前收尾**：离开前更新「当前状态 / 下一步」段落，确保下一棒知道从哪接。
5. **不破坏现有契约**：`store.js` / `user.js` / `room.js` 对外接口保持兼容；UI 铁律（禁 emoji 当图标、用 t-icon）不变。
6. **真机验证**：涉及实时同步/推送的功能，需两台手机在微信开发者工具里验证（云开发环境已在工具内选定）。

> 这条规则**对所有未来任务生效**，不仅限本项目。已写入 `CLAUDE.md`（项目）与我的长期记忆。
